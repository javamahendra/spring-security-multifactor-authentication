+## API Overview
+
+Authentication
+- POST `/api/auth/register`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/auth/register \
+         -H "Content-Type: application/json" \
+         -d '{"username":"john","password":"secret","role":"USER"}'
+    ```
+  - Response (on success): `AuthenticationResponse` with JWT tokens or an MFA challenge if MFA already enabled.
+
+- POST `/api/auth/login`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/auth/login \
+         -H "Content-Type: application/json" \
+         -d '{"username":"john","password":"secret"}'
+    ```
+  - Responses:
+    - If MFA NOT enabled:
+      ```json
+      {"accessToken":"...","refreshToken":"...","mfaRequired":false,"mfaVerified":true}
+      ```
+    - If MFA enabled (challenge):
+      ```json
+      {"mfaRequired":true,"mfaVerified":false,"mfaToken":"<temporary-mfa-token>"}
+      ```
+
+- POST `/api/auth/refresh`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/auth/refresh \
+         -H "Content-Type: application/json" \
+         -d '{"refreshToken":"<your-refresh-token>"}'
+    ```
+  - Response: new `accessToken` issued; `refreshToken` retained if valid.
+
+- POST `/api/auth/logout`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/auth/logout \
+         -H "Authorization: Bearer <your-access-token>"
+    ```
+  - Invalidates all tokens for the authenticated user in Redis.
+
+MFA Setup & Confirmation (while logged in)
+- POST `/api/mfa/setup`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/mfa/setup \
+         -H "Authorization: Bearer <your-access-token>"
+    ```
+  - Response:
+    ```json
+    {
+      "secret":"BASE32SECRET",
+      "qrCodeUri":"otpauth://totp/App:john?secret=...",
+      "qrCodeImage":"data:image/png;base64,iVBOR..."
+    }
+    ```
+  - Scan `qrCodeImage` (or use `qrCodeUri`) in Google Authenticator.
+
+- POST `/api/mfa/confirm`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/mfa/confirm \
+         -H "Authorization: Bearer <your-access-token>" \
+         -H "Content-Type: application/json" \
+         -d '{"code":"123456"}'
+    ```
+  - Response: `200 OK` on success and the user is forced to re‑login. MFA becomes active for subsequent logins.
+
+MFA Verification after Login Challenge
+- POST `/api/auth/verify/mfa`
+  - Request:
+    ```bash
+    curl -X POST http://localhost:8080/api/auth/verify/mfa \
+         -H "Content-Type: application/json" \
+         -d '{"mfaToken":"<temporary-mfa-token>","code":"123456"}'
+    ```
+  - Response (on success):
+    ```json
+    {"accessToken":"...","refreshToken":"...","mfaRequired":false,"mfaVerified":true}
+    ```
+
+## Security Notes
+- MFA secret is generated per user and stored encrypted using AES‑GCM via `EncryptionService`.
+- Temporary MFA tokens are stored in Redis with short TTL and removed after successful verification.
+- JWT validation and token blacklisting/rotation are handled by `JwtTokenProvider` and `TokenRepository`.
